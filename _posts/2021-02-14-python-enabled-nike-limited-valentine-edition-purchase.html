Firstly, I figured out if there's any difference between the pages of products available right now and the products yet to be launched!

The visible difference is the text Coming Soon not being present in the product which is in stock.

In HTML terms, this translated into a div with data-text="comingSoon"` not being present when product is in stock.

All I needed was a script which continuously scraped the page and check for the tag being present. The moment the product comes in stock, the tag would go away and in such case, a notification needed to be sent.

Wrote a python script to do the same.

The script makes use of two external services

Call Me Bot for sending whatsapp messages.

Wirepusher for sending mobile notifications.

The links have detailed description on how to set up both of them.

This script now had to be run on a machine and had to be run on regular intervals i.e. a cron had to be setup.

I chose GCP to be the cloud platform. You could follow this article to get it done.

Once the machine was up, I cloned the repository.

Only this left was to get the API Keys and set up a cron which ran every 2 minutes.

*/2 * * * * env MOBILE_NUMBER="#mobile_number" CALL_ME_BOT_KEY="#call_me_bot_key" WIRE_PUSHER_KEY="#wire_pusher_key" python /scripts-py/nikeshoe_alerter.py >> /log.py
